## 🔍 根因分析 (Root Cause)

经过完整代码审查，摘要功能无法工作有 **两个根本原因**：

### 问题 1：模型文件未部署到设备 ❌
- `MediaPipeLlmSummaryGenerator` 在设备上查找 `files/models/gemma3-1b-it-int4.task`
- **`models/` 目录为空** — 模型从未成功下载和推送
- 没有模型文件 → `modelExists()` 返回 false → `isAvailable()` 返回 false
- `SummaryManager` fallback 到 `TextRankSummaryGenerator`（纯算法抽取，非 LLM）

### 问题 2：16KB 对齐导致 APP 无法安装 ❌
- Android 16 (API 36) **强制要求** 所有 native .so 库 16KB 页对齐
- `libllama_jni.so` — **已废弃的旧 llama.cpp 构建产物**，C++ 源码已删除但 `build/` 目录残留
- `libandroidx.graphics.path.so` — 来自 Compose BOM 间接依赖，版本过旧未 16KB 对齐
- `libllm_inference_engine_jni.so` — MediaPipe `tasks-genai:0.10.22` 的 native 库，可能未 16KB 对齐

### 流程链路：
```
保存笔记 → NoteViewModel.insertNote() 
  → SummaryManager.generateSummaryForNote()
    → checkAvailability()
      → MediaPipeLlmSummaryGenerator.isAvailable() → ❌ 模型文件不存在
      → TextRankSummaryGenerator.isAvailable() → ✅ 始终可用
    → activeGenerator = TextRankSummaryGenerator (纯算法，非LLM)
    → generateSummary() → TextRank 抽取式摘要 (非真正 LLM 生成)
```

## 🤖 模型选型

### 最终选择：**Gemma 3 1B IT int4** (MediaPipe .task 格式)

| 维度 | 详情 |
|---|---|
| 模型 | Gemma 3 1B Instruct (Google) |
| 量化 | int4 (4-bit 量化) |
| 文件大小 | **~550 MB** |
| 运行时内存 | ~1-2 GB |
| 推理框架 | MediaPipe LLM Inference (已集成) |
| 下载地址 | `https://storage.googleapis.com/mediapipe-models/llm_inference/gemma3_1b_it_int4/float32/latest/gemma3-1b-it-int4.task` |
| 格式 | `.task` (MediaPipe 原生格式，开箱即用) |

### 选型理由：
1. **代码已就绪** — `MediaPipeLlmSummaryGenerator.kt` 已完整实现，只缺模型文件
2. **内存充足** — 设备 12GB，可用 3.4GB，模型运行时 ~1-2GB，完全足够
3. **存储充足** — 26GB 可用，550MB 模型轻松放下
4. **质量优秀** — 1B 参数，支持中英文，摘要质量远超 TextRank
5. **Google 官方维护** — MediaPipe 原生支持，API 稳定
6. **已有框架集成** — 不需要重新写推理代码

### 对比其他方案（已排除）：

| 方案 | 排除原因 |
|---|---|
| Gemini Nano (AI Core) | 之前尝试失败，需要 Google 账号加入 beta，不稳定 |
| llama.cpp + GGUF | 之前尝试过，16KB 对齐问题严重，需要编译 native 代码，复杂度高 |
| ONNX Runtime | 需要额外转换模型格式，集成复杂 |
| SmolLM / Phi-3 | 需要另外集成推理引擎，工作量大 |

## 📋 修复步骤 (4 步)

### Step 1: 清理构建产物，消除 16KB 对齐问题
**目标**: 移除残留的 `libllama_jni.so`，更新依赖修复 16KB 对齐

```bash
# 1. 清理整个 build 目录（移除残留 libllama_jni.so）
cd /Users/wangkun/AndroidStudioProjects/Notebook
./gradlew clean

# 2. 删除 cxx 缓存（防止 CMake 残留）
rm -rf app/build app/.cxx
```

**依赖更新 (gradle/libs.versions.toml)**:
```toml
# 更新 Compose BOM 到 2025.05.01 (修复 graphics-path 16KB 对齐)
composeBom = "2025.05.01"

# 更新 MediaPipe 到最新版 (修复 libllm_inference_engine_jni.so 16KB 对齐)
mediapipeGenai = "0.10.24"
```

**备选方案** — 如果更新依赖后仍有 16KB 问题：
在 `build.gradle.kts` 中添加 `packagingOptions` 排除有问题的 ABI：
```kotlin
android {
    packagingOptions {
        jniLibs {
            // 只保留 arm64-v8a (Pixel 8 Pro 的架构)
            // 减少有问题的 .so 文件
        }
    }
}
```

### Step 2: 下载模型并推送到设备
**目标**: 将 Gemma 3 1B IT int4 模型部署到 Pixel 8 Pro

```bash
# 1. 下载模型到本地 (约 550MB)
cd /Users/wangkun/AndroidStudioProjects/Notebook
mkdir -p models
curl -L --progress-bar \
  -o models/gemma3-1b-it-int4.task \
  "https://storage.googleapis.com/mediapipe-models/llm_inference/gemma3_1b_it_int4/float32/latest/gemma3-1b-it-int4.task"

# 2. 推送到设备临时目录
adb push models/gemma3-1b-it-int4.task /data/local/tmp/

# 3. 先安装更新后的 APP（Step 3 完成后）

# 4. 拷贝模型到 APP 私有目录
adb shell "run-as com.example.notebook mkdir -p files/models"
adb shell "cp /data/local/tmp/gemma3-1b-it-int4.task /data/data/com.example.notebook/files/models/"
adb shell "run-as com.example.notebook chmod 644 files/models/gemma3-1b-it-int4.task"

# 5. 验证
adb shell "run-as com.example.notebook ls -la files/models/"

# 6. 清理临时文件
adb shell "rm /data/local/tmp/gemma3-1b-it-int4.task"
```

### Step 3: 重新构建并安装 APP
**目标**: 构建不含 `libllama_jni.so` 的干净 APK

```bash
./gradlew :app:assembleDebug
adb install -r app/build/outputs/apk/debug/app-debug.apk
```

**验证**: 检查 APK 中不再包含 `libllama_jni.so`：
```bash
unzip -l app/build/outputs/apk/debug/app-debug.apk | grep "\.so"
# 应只看到: libllm_inference_engine_jni.so, libandroidx.graphics.path.so
# 不应看到: libllama_jni.so
```

### Step 4: 验证摘要功能
**目标**: 确认 LLM 摘要生成端到端工作

1. 启动 APP → 不应出现 16KB 对齐报错
2. 创建新笔记，输入 50+ 字内容
3. 保存 → 观察 SummaryCard 状态变化: PENDING → GENERATING → COMPLETED
4. 查看 Logcat 确认调用路径:
```bash
adb logcat -s "MediaPipeLlm" "SummaryManager"
# 期望看到:
# SummaryManager: AI availability: true, active=MediaPipeLlmSummaryGenerator
# MediaPipeLlm: LLM ready in xxx ms
# MediaPipeLlm: inference xxx ms  raw=xxx
# SummaryManager: Summary generated by: MediaPipeLlmSummaryGenerator
```
